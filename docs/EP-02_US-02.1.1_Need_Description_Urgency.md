## US-02.1.1 — Need Description, Urgency and Official Documentation

**Epic:** EP-02 — Need Description and Baseline Scenario
**Feature:** F-02.1 — Need Characterisation
**Priority:** Must Have
**Dependencies:** US-01.3.1 (a need must be selected in Phase 1)

### 1. What this screen/component does
Step 2.1 page ("Fabbisogno" in the sidebar). Presents three input sections in a single page for the officer to characterise the territorial need: a free-text description, an urgency assessment, and an optional reference to official acts. The data is descriptive — it feeds the DOCFAP report narrative and does not participate in any calculation.

### 2. Purpose
Allow the officer to articulate the public need in their own words, declare how urgent the intervention is, and optionally anchor it to existing administrative acts. This satisfies D.Lgs. 36/2023, Annex I.7, Art. 2, c.2 — explicit identification of the need that justifies the investment.

### 3. Input data

| Field | Type | Required | Source |
|---|---|---|---|
| Need description | String (textarea) | Yes | User input |
| Urgency | Single select (radio) | Yes | Fixed frontend options (4 levels) |
| Documented in official acts | Single select (radio) | No | Fixed frontend options (3 options) |
| Act reference | String (text input) | No | User input (conditional) |

### 4. Pre-populated or default data
- **Need description:** empty on first access. The helper text displays the `description` field of the need selected in Step 1.3 (read from OpenCore Layer 1 → Needs) as contextual guidance, not as a pre-filled value.
- **Urgency:** no pre-selection on first access.
- **Documented in official acts:** no pre-selection on first access.
- **Act reference:** hidden until the "Documented" field is set to "Sì" or "Parzialmente".
- If the user returns to this step, all fields are restored from the store.

### 5. Business rules and validations
- **Need description:** required. Must not be blank after trimming. Maximum 600 characters. A visible character counter shows remaining characters.
- **Urgency:** required. Exactly one option must be selected.
- **Documented in official acts:** optional. If left unselected the step can still advance.
- **Act reference:** appears only when "Documented" is "Sì" or "Parzialmente". Always optional. If the user changes "Documented" to "No", the reference field is hidden and its value is cleared.
- **Advancement rule:** the step can advance only when the description is non-empty AND urgency is selected.

### 6. UI element behaviour

| Element | Type | Initial state | Behaviour | Effect |
|---|---|---|---|---|
| Need description | Textarea (max 600 chars) | Empty | User types; character counter updates | Updates store `problema.descrizione` |
| Helper text | Static text below label | Shows `description` from selected need | Read-only | — |
| Character counter | Inline counter | "600 characters remaining" | Decrements as user types | — |
| Urgency | Radio group (4 options) | No selection | User selects one | Updates store `urgenza` |
| Emergency alert | Inline alert (warning) | Hidden | Visible when urgency = "Emergenziale (entro 12 mesi)" | Read-only informational |
| Documented in official acts | Radio group (3 options) | No selection | User selects one | Updates store `problema.documentato` |
| Act reference | Text input | Hidden | Appears when documented = "Sì" or "Parzialmente"; user types | Updates store `problema.riferimentoAtto` |
| "Next" button | Navigation button | Disabled | Active when description non-empty AND urgency selected | Navigates to Step 2.2 |

#### Urgency options (fixed, frontend)

| Value | Label |
|---|---|
| `emergenziale_12m` | Emergenziale (entro 12 mesi) |
| `breve_1_3` | Breve termine (1-3 anni) |
| `medio_3_5` | Medio termine (3-5 anni) |
| `strutturale` | Strutturale/programmato |

#### "Documented in official acts" options (fixed, frontend)

| Value | Label |
|---|---|
| `si` | Sì |
| `parzialmente` | Parzialmente |
| `no` | No |

```
User fills in need characterisation fields
  → Local state update (store)
  → No API call
```

### 8. OpenCore entities involved

| Entity | Table | Usage |
|---|---|---|
| Need | Layer 1 → Needs | `description` field read for helper text |

### 9. Edge cases and conditional behaviour
- **If the user clears the description after typing:** the step cannot advance. Error message shown on blur: "La descrizione del fabbisogno è obbligatoria."
- **If urgency is "Emergenziale":** an inline alert appears with text: "Verificare compatibilità con procedure ordinarie D.Lgs. 36/2023 art. 140 e seguenti." The alert has `role="alert"` and `aria-live="assertive"`.
- **If "Documented" changes from "Sì"/"Parzialmente" to "No":** the act reference field disappears and its stored value is cleared to empty string.
- **If the user returns to this step:** all fields are restored from the store, including conditional visibility of the act reference field.
- **If the need was changed in Phase 1 after this step was already filled:** the description helper text updates to reflect the new need's `description`. The user's typed description is NOT cleared — they may have customised it independently.

### 10. Acceptance Criteria

```gherkin
Scenario: Successful need characterisation
  Given the officer is on Step 2.1
  When they enter a non-empty description
  And they select an urgency level
  Then the "Next" button becomes active

Scenario: Description required
  Given the officer is on Step 2.1
  When the description field is empty
  Then the step cannot advance
  And an error is shown on the description field

Scenario: Urgency required
  Given the officer is on Step 2.1
  When no urgency level is selected
  Then the step cannot advance

Scenario: Emergency alert
  Given the officer is on Step 2.1
  When they select "Emergenziale (entro 12 mesi)"
  Then an inline warning alert about D.Lgs. 36/2023 art. 140 is displayed

Scenario: Conditional act reference
  Given the officer is on Step 2.1
  When they select "Sì" or "Parzialmente" for documented in official acts
  Then the act reference text field appears
  And it is optional

Scenario: Act reference cleared on "No"
  Given the officer selected "Sì" and entered an act reference
  When they change "Documented" to "No"
  Then the act reference field is hidden
  And the stored value is cleared

Scenario: Documentation is optional
  Given the officer has filled description and urgency
  When "Documented in official acts" has no selection
  Then the step can still advance
```
