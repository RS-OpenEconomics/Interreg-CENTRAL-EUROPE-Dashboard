## US-02.2.1 — Scenario Zero Structured Questions

**Epic:** EP-02 — Need Description and Baseline Scenario
**Feature:** F-02.2 — Scenario Zero Narrative
**Priority:** Must Have
**Dependencies:** US-01.3.1 (a need must be selected)

### 1. What this screen/component does
Step 2.2 page ("Scenario zero" in the sidebar). On entry, the system loads the Scenario Zero context questions associated with the selected need. The officer answers all questions to build a qualitative narrative of the current situation (the baseline "what happens if we do nothing" scenario). Each answer carries a text fragment that is assembled into a prose paragraph for the DOCFAP report.

If no questions are configured for the selected need, this component does not render — see US-02.2.2 (Free-Text Fallback) instead.

### 2. Purpose
Generate the "Analisi dello Scenario di Riferimento" section required by D.Lgs. 36/2023, Annex I.7, Art. 2, c.4(a). The Scenario Zero describes the baseline situation and the projected consequences of non-intervention. This is a purely narrative output — it does not produce scores, activate KPIs, or participate in any quantitative analysis.

### 3. Input data

| Field | Type | Required | Source |
|---|---|---|---|
| Scenario Zero questions | SZContextQuestion[] | — | OpenCore Layer 1 → `sz_domande_contesto` (filtered by `fabbisogno_code` = current `fabId`) |
| Answers (per question) | string (radio) or string[] (checkbox) | Yes (all) | User selection |

### 4. Pre-populated or default data
- Questions are loaded from the API on step entry, filtered by the current `fabId`.
- If the user returns to this step, previous answers are restored from the store.
- On first access, no answers are pre-selected.

### 5. Business rules and validations
- **All questions are mandatory.** For radio questions: exactly one option must be selected. For checkbox questions: at least one option must be selected.
- The "Next" button is disabled until every question has a valid answer.
- If the user attempts to advance with incomplete answers, an error message is shown for each unanswered question.
- **Narrative assembly:** on every answer change, the frontend assembles the narrative text by substituting `text_fragment` values into the `narrative_template` of the corresponding `ZeroScenarioCategory`:
  - For radio questions: the selected answer's `text_fragment` replaces the `{DC-SZ-NNN-NN}` placeholder.
  - For checkbox questions: the `text_fragment` values of all selected answers are joined with `", "` and the result replaces the placeholder.
- The assembled text is saved to the store as `scenarioZeroNarrative`.
- The answer map is saved to the store as `scenarioZeroAnswers` (keyed by `questionId`).

### 6. UI element behaviour

| Element | Type | Initial state | Behaviour | Effect |
|---|---|---|---|---|
| Loading spinner | Spinner + sr-only text | Visible during fetch | Disappears when questions are loaded | — |
| Question fieldset | Fieldset with legend | No selection | User interacts with radio/checkbox | Updates `scenarioZeroAnswers`; triggers narrative rebuild |
| Radio option | Radio input inside fieldset | Unchecked | User selects one per question | Saves answer; rebuilds narrative |
| Checkbox option | Checkbox input inside fieldset | Unchecked | User selects one or more | Saves answer array; rebuilds narrative |
| Validation error | Inline error per fieldset | Hidden | Shown on advance attempt if question unanswered | — |
| "Next" button | Navigation button | Disabled | Active when all questions answered | Navigates to Step 2.3 |

### 7. Frontend → OpenCore flow

```
Step 2.2 loads
  → GET Scenario Zero questions for fabId
     API: Layer 1 → sz_domande_contesto (filter: fabbisogno_code = fabId)
  → Returns: SZContextQuestion[] (1–5 questions, ordered by `order` field)
  → Render questions

User answers questions
  → Local state update: scenarioZeroAnswers, scenarioZeroNarrative
  → No API call for narrative assembly (frontend-only text substitution)
```

### 8. OpenCore entities involved

| Entity | Table | Usage |
|---|---|---|
| SZContextQuestion | Layer 1 → `sz_domande_contesto` | Questions and answer options for the selected need |
| ZeroScenarioCategory | Layer 3 → `zero_scenario_categories` | `narrative_template` used for text assembly |

### 9. Edge cases and conditional behaviour
- **Loading state:** during the API fetch, a spinner is shown with accessible text "Caricamento domande in corso". No questions are rendered until the fetch completes.
- **API error:** if the question fetch fails, an error message is shown with a retry button.
- **Need change after answering:** if the user navigates back to Phase 1 and changes the selected need, all Scenario Zero answers and the assembled narrative are cleared from the store.
- **Variable number of questions:** the number of questions varies by need (1 to 5). The UI renders dynamically — no fixed layout assumption.

### 10. Acceptance Criteria

```gherkin
Scenario: Questions loaded for selected need
  Given the officer enters Step 2.2
  And the selected need has 3 Scenario Zero questions configured
  When the questions are loaded
  Then 3 fieldsets are displayed, each with the question text as legend

Scenario: All questions mandatory
  Given 3 questions are displayed
  When the officer has answered only 2
  Then the "Next" button remains disabled

Scenario: Radio question answered
  Given a radio question is displayed
  When the officer selects one option
  Then the answer is saved in the store
  And the narrative is rebuilt with the selected text_fragment

Scenario: Checkbox question answered
  Given a checkbox question is displayed
  When the officer selects 2 options
  Then both answers are saved in the store
  And the narrative contains both text_fragments joined with ", "

Scenario: Advance with all questions answered
  Given all questions have valid answers
  When the officer clicks "Next"
  Then they advance to Step 2.3
  And scenarioZeroNarrative contains the fully assembled text

Scenario: Validation errors on incomplete advance
  Given 1 question has no answer
  When the officer attempts to advance
  Then an error message is shown on the unanswered question's fieldset
```
