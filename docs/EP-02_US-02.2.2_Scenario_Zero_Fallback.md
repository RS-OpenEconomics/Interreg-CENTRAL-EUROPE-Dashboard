## US-02.2.2 — Scenario Zero Free-Text Fallback

**Epic:** EP-02 — Need Description and Baseline Scenario
**Feature:** F-02.2 — Scenario Zero Narrative
**Priority:** Must Have
**Dependencies:** US-02.2.1

### 1. What this screen/component does
Alternative rendering of Step 2.2 when the selected need has no Scenario Zero questions configured in OpenCore. Instead of structured questions, the system displays a single textarea where the officer describes the current situation in free form.

### 2. Purpose
Guarantee that every DOCFAP produces a Scenario Zero narrative, even for needs not yet covered by the structured question catalogue. The free-text fallback ensures the officer can always proceed, satisfying D.Lgs. 36/2023, Annex I.7, Art. 2, c.4(a).

### 3. Input data

| Field | Type | Required | Source |
|---|---|---|---|
| Scenario Zero narrative | String (textarea) | Yes | User input |

### 4. Pre-populated or default data
- If the user returns to this step, the previously entered text is restored from the store.

### 5. Business rules and validations
- The textarea is mandatory — cannot be blank after trimming.
- No maximum character limit (the text goes directly into the report section).
- The entered text is saved to the store as `scenarioZeroNarrative`.
- `scenarioZeroAnswers` remains an empty object (no structured answers).

### 6. UI element behaviour

| Element | Type | Initial state | Behaviour | Effect |
|---|---|---|---|---|
| Textarea | Textarea (resizable) | Empty (or restored from store) | User types | Updates store `scenarioZeroNarrative` |
| Label | Static label | "Descrivi la situazione attuale rispetto al fabbisogno" | Read-only | — |
| "Next" button | Navigation button | Disabled | Active when textarea non-empty | Navigates to Step 2.3 |

### 7. Frontend → OpenCore flow

```
Step 2.2 loads
  → GET Scenario Zero questions for fabId → returns empty array
  → Render free-text fallback

User types narrative
  → Local state update: scenarioZeroNarrative
  → No API call
```

### 8. OpenCore entities involved

None directly. The text is saved as-is in the store.

### 9. Edge cases and conditional behaviour
- **Trigger condition:** the fallback renders when the API returns zero questions for the current `fabId`. This is the only trigger — there is no manual toggle.
- **If the user clears the textarea after typing:** the step cannot advance. Error message on blur: "La descrizione dello scenario zero è obbligatoria."
- **If the need changes and the new need has structured questions:** the free-text is cleared and the structured view (US-02.2.1) takes over.

### 10. Acceptance Criteria

```gherkin
Scenario: Fallback rendered for uncovered need
  Given the selected need has no Scenario Zero questions
  When Step 2.2 loads
  Then a single textarea with the label "Descrivi la situazione attuale rispetto al fabbisogno" is displayed
  And no fieldsets are shown

Scenario: Free text is mandatory
  Given the fallback textarea is displayed
  When the textarea is empty
  Then the step cannot advance

Scenario: Successful free-text entry
  Given the fallback textarea is displayed
  When the officer enters text
  Then scenarioZeroNarrative is updated in the store
  And the "Next" button becomes active
```
